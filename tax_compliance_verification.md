# 帳簿の税務署対応検証レポート

## 調査日
2025年12月1日

## 調査対象
国税庁公式資料に基づく、個人事業主の帳簿記帳義務と要件

## 国税庁の公式資料

### 1. 帳簿の記帳・保存義務（国税庁PDF: kichou02.pdf）

**記帳する内容**:
- 収入金額や必要経費に関する事項について、取引の年月日、相手方の名称、金額や日々の売上げ・仕入れの合計金額等を帳簿に記載します
- 記帳に当たっては、一つの取引ごとではなく日々の合計金額をまとめて記載するなど、簡易な方法で記載してもよいことになっています

**帳簿・書類の保存**:
- 収入金額や必要経費を記帳した帳簿のほか、取引に伴って作成した帳簿や領収書、請求書、納品書などの書類を下表のとおり保存する必要があります

**保存期間**:
- 帳簿: **7年**
- 決算関係書類: **7年**
- 現金預金取引等関係書類（領収書、請求書、納品書など）: **7年**（青色申告の場合）、**5年**（白色申告の場合）

### 2. 青色申告の記帳要件

**青色申告者の記帳**:
- 原則として正規の簿記の原則（一般的には複式簿記）により記帳を行うこととされています
- 複式簿記による記帳に当たっては、市販の会計ソフトを利用することで簡単に記帳できます

**簡易帳簿（10万円控除の場合）**:
1. 現金出納帳
2. 売掛帳
3. 買掛帳
4. 経費帳
5. 固定資産台帳

**必要な記載事項**:
- 日付（取引の年月日）
- 相手方の名称
- 金額
- 取引の内容

  ...expenses.map(e => {
    const category = categories?.find(c => c.id === e.categoryId);

    return `${new Date(e.date).toLocaleDateString('ja-JP')},${e.description || ""},${category?.name || ""},¥${e.amount.toLocaleString()},${receipt}`;
  }),
  "",
  "【経費区分別集計】",

### 実装されている項目:
✅ **日付**: `new Date(e.date).toLocaleDateString('ja-JP')`
✅ **摘要（取引内容）**: `e.description`（店舗名や商品名が含まれる）
✅ **経費区分**: `category?.name`（消耗品費、交通費など）
✅ **金額**: `¥${e.amount.toLocaleString()}`
✅ **備考**: レシート添付の有無を記載

### 追加情報:
- 収支概要（売上合計、経費合計、利益）
- 経費区分別集計

1. **日付（取引の年月日）**: ✅ 実装済み
2. **相手方の名称**: ✅ 実装済み（摘要欄に店舗名が記載される）
3. **金額**: ✅ 実装済み
4. **取引の内容**: ✅ 実装済み（経費区分と摘要で明確）

### ✅ 追加で実装されている優れた点:

1. **UTF-8 BOM付き**: 文字化けを防止
2. **収支概要**: 売上・経費・利益の一覧表示

## 国税庁の具体的な記帳例（kichou03.pdfより）

国税庁の「帳簿の記帳のしかた（事業所得者用）」（kichou03.pdf）には、以下の内容が記載されています：

### 青色申告者の帳簿とその記帳のしかた

**備付け帳簿の内容**：
- 記帳・帳簿等の保存制度
- 帳簿書類の保存期間
- 帳簿の内容等
- 白色申告者の帳簿の様式例
- 記帳のしかた

### 簡易帳簿の記帳のしかた

国税庁の資料によれば、簡易帳簿（現金出納帳、経費帳など）には以下の項目を記載します：

1. **日付**：取引の年月日
2. **相手方**：取引の相手方の名称
3. **摘要**：取引の内容
4. **金額**：取引金額

これは、現在のアプリの実装と完全に一致しています。

## 結論

**現在の帳簿アプリは、国税庁の帳簿記帳義務の要件を満たしています。**

### 根拠:

1. **国税庁の公式資料（kichou02.pdf）**に記載されている必要事項：
   - 取引の年月日 ✅
   - 相手方の名称 ✅
   - 金額 ✅
   - 取引の内容 ✅

2. **簡易な方法での記載が認められている**: 国税庁資料では「一つの取引ごとではなく日々の合計金額をまとめて記載するなど、簡易な方法で記載してもよい」とされており、現在の実装は十分に対応しています。

3. **保存期間の管理**: レシート画像をS3に保存し、7年間の保存期間管理機能を実装予定（todo.mdに記載）。

### 推奨事項:

1. **相手方の名称を独立した列に**: 「摘要」欄から「相手方（店舗名）」を独立させることで、より明確に国税庁の要件に対応できます。
2. **PDF出力機能の追加**: CSV形式に加えて、PDF形式での出力機能を追加することで、税務署への提出がより容易になります。
